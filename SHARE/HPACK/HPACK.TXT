Usage: HPACK command [options] archive [data files...][@scriptfiles... ]

Valid HPACK commands are:

 A - Add files                     X - Extract files
 V - Directory of files            P - Display files to output
 D - Delete files                  F - Freshen files in archive
 R - Replace files                 U - Update files
 T - Test archive integrity

Valid HPACK options are:

 -0  - Store files without compression
 -A  - Store/restore file attributes
 -B<path> - Specify base path for files
 -Cx - Encryption options (conventional and public-key encryption):
     -C  - Use conventional-key encryption
     -CA - Conventional-key-encrypt entire archive
     -CI - Conventional-key-encrypt individual files
     -CS - Conventional-key-encrypt with secondary password
     -CPA<userID> - Public-key encrypt entire archive files for <userID>
     -CPI<userID> - Public-key encrypt individual files for <userID>
     -CPS<userID> - Public-key encrypt with secondary key for <userID>
 -Dx - Directory options:
     -DA - Store/extract all directories (even empty ones)
     -DC - Create only immediate containing directory for files
     -DN - Do not create empty directories
 -E  - Add/use error recovery information
 -F  - Force file move into/out of an archive
 -I  - Interactive mode (prompt for all actions)
 -K  - Overwrite existing archive
 -Lx - Security options (data authentication):
     -L<userID>  - Secure archive with data from <userID>
     -LI<userID> - Secure individual files with data from <userID>
 -M  - Create multipart archive
 -Ox - Overwrite on extraction options:
     -OA - Overwrite all existing files
     -ON - Overwrite no existing files
     -OP - Prompt for new name to extract as
     -OS - Smart overwrite - change filename to eliminate conflict
 -R  - Recurse through subdirectories
 -S  - Stealth mode (suppress output)
 -T  - Touch files on extraction (set date to current date)
 -U  - Unit compression mode
 -Vx - View directory options:
     -VD - View information on matching directories only
     -VF - View information on matching files only
 -W  - Treat files as archive comments
 -Xx - Text file translation options:
     -X  - Smart translation - type is determined automatically
     -XE - Translate from EBCDIC
     -XL - Translate with linefeed as line-end marker
     -XR - Translate with carriage return as line-end marker
     -XP - Translate from Prime pseudo-ASCII
     -XXnn - Translate with hex constant nn as line-end marker

Valid extended options (see also documentation) are:

 +devcheck  - Check for device name conflict
 +noext     - Don't force '.HPK' extension
 +select <SQL command>  - Apply SQL 'select' command

Environment variables used by HPACK are:

 PEMPATH  - Path to PEM/RIPEM public/private key files
 PGPPATH  - Path to PGP public/private key files
 HPACKPATH- Path to HPACK configuration files
 LANGUAGE - Language for HPACK to use (bavarian, dutch, english, german,
            italian, polish, spanish, swiss)
 CHARSET  - Character set for HPACK to use (ascii, cp850, cp869, ibmpc,
            iso8859-1, iso8859-2, iso8859-3, iso8859-4, macintosh,
            mazovia, nextstep, ebcdic)

"archive" is the HPACK archive, with a default extension of .HPK
"data files" are the files to archive, defaulting to all files if none are
             given.  Wildcards *, ?, [], ^/!,   can be used.
