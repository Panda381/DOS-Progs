Example JAR commands:
---------------------

Add *.exe files with directory structure:  JAR a -r archive *.exe
Add two files to archive:                  JAR a archive name1 name2
Add files to archive with verification:    JAR a -jt archive *.*
Add files with maximum compression and
archive lock:                              JAR a -m4 -jl archive *
Comment archive header only:               JAR c archive -zcmt.fil
Strip archive comment only:                JAR c archive -zNUL
Extract files to directory "c:\textfile":  JAR e archive -oc:\textfile
Extract maintaining directory structure:   JAR x archive
Extract subdirectory from archive:         JAR e archive subdir\*
List files in archive:                     JAR l archive
Move files to archive:                     JAR m archive *.doc
Test integrity of files in archive:        JAR t archive
Add files to a multiple volume archive:    JAR a -va a:archive *.*
Add new chapter and comment:               JAR ac archive *.doc -hz
List chapters:                             JAR lc archive
Delete few chapters:                       JAR dc archive 1-5
Split archive:                             JAR y archive -v100k -oa:split
Extract from first chapter:                JAR e archive -jb1

To run the Win32 JAR version you need to specify "JAR32" instead of "JAR".

Usage:
------

   JAR <command> [<switch>...] <archive_name>[.j] [<arguments>...]

   <arguments>::= <switch> | <wildcard> | <filename> | <list_file> |
                  <response_file>
   <switch>::= {'/'|'-'}<switch_character>[-|<option>]
   <list_file>::= '!'<filename>
   <response_file>::= '@'<filename>

Commands:
---------

    a: Add files to archive                 m: Move files to archive
   ac: Add new Chapter                    reg: Register JAR and JAR32
    c: Comment archive files                   (DOS version only)
  cfg: Extract default JAR.CFG file        rh: Recover archive Header
    d: Delete files from archive               (Win32 version only)
   da: Delete files in All chapters        sl: Scan Locked files
   dc: Delete Chapters                      t: Test integrity of archive
    e: Extract files from archive        test: Test JAR reliability
    f: Freshen files in archive             u: Update files to archive
    l: List contents of archive             v: Verbosely list archive contents
   lc: List Chapters                        x: eXtract files with full pathname
   ls: List (Short) contents of archive     y: copY archive with new options
   lt: List Technical information

User prompt responses:
---------------------

Yes      - yes
No       - no
Quit     - abort out of JAR
Always   - always assume yes for current type of query
Skip     - always assume no for current type of query
Global   - always assume yes for all queries except diskette volume prompts
Command  - prompt for and execute one DOS command

Switches:
---------

   b: set/reset Archive bits               v: enable multiple Volumes
      b0: Reset                               v360: build 362000 byte volumes
      b1: Set                                 v50K: build 50000 byte volumes
   d: Delete added files, asks                va: auto-detect space available
      permission before deleting              vi: inhibit volume change test
   f: Freshen existing files                  vr50K: reserve 50000 bytes of
      f1: Freshen with older files                   space on first volume
      f2: Freshen with different files        vs: provide DOS command prompt
   g: specify archive password                vsCMD: execute CMD before each
      gmike1234: use password mike1234               volume
      g?: prompt for password                 vv: beep between volumes
   l: create List_name file                   vz: provide command with no echo
      lNAMES.LST: create NAMES.LST            v360,v720,v1200,v1440: abbrevs
   m: with Method 1, 2, 3, 4                  Volume options may be in any
      m1: fast compression                    order except s and z which must
      m2: normal compression                  be the last option.
      m3: better compression               w: assign Work directory
      m4: best compression                    wC:\TMP: use C:\TMP as work dir
   n: only New files (not exist)           x: eXclude filenames
   o: set Output directory/file               x*.EXE: exclude *.EXE files
   q: Query on each file                      x!NAMES: exclude files in NAMES
   r: Recurse subdirectories                  Multiple exclusions are allowed
   s: archive time-Stamp                   y: assume Yes on all queries
      s: set to newest                        except diskette volume prompts.
      s1: save original                       Use this switch for batch mode
   u: Update files (new + newer)           z: supply file for archive comment
      u1: Update files (new + older)          zA.CMT: use A.CMT for comments
      u2: Update files (new + different)      zNUL: use to strip comments

Shifted switches:
-----------------

   jb: select chapter or chapter range     ha: ignore readonly Attribute
       jb10:   chapter 10                  hb: select files by attributes
       jb1-20: chapters 1-20                   a - archive
       jb20-:  chapters 20 and above           d - directory
       jb-10:  chapters 1-10                   h - hidden
       jb1-:   all chapters                    r - readonly
   jf: store Full path                         s - system
   jg: specify new password during copy        0 - select bits = 0
       jgmike1234: set new archive             1 - select bits = 1
                   password to mike1234        hbsh0r1: select readonly but
       jg?: prompt for new password                not system or hidden files
       jg:  remove password                he: skip test of security Envelope
   jl: set Lock option                     hf: long Filenames processing
       jl:  Lock archive                       hf-: match long names (default)
       jl1: Lock existing chapters             hf:  match long and short names
       jl2: make auto-Lockable archive         hf1: use only short Filenames
   jr: Recover broken archive files            hf2: convert long Filenames
       (Win32 version only)                         into short names
   js: Select chapter creation mode            hf3: convert long Filenames
       js-: compress all files                      into short names without
       js0: compress file if creation               international characters
            or modification time or size   hg: set access protection option
            changed (default)                  hg0: authentication
       js1: compress file if modification      hg1: encryption
            time or size changed           hk: add recovery records
   jt: Test temporary archive              hl: warn of unused wildcards
   ju: sort files while listing            hm: On or after YYMMDDHHMMSS
       ju-: do not sort                        hm:       on today
       ju0: sort files by name                 hm960125: on/after 01/25/96
       ju1: sort files by extension        hmb: Before YYMMDDHHMMSS
   jx: hide archive contents                    hmb:       before today
   jy: suppress queries assuming Yes            hmb960125: before 01/25/96
       c - skip create directory query     hmd: no older than N Days
       d - skip delete files query              hmd5: five or less days old
       k - skip disk space available       ho: Output directory translation
           query                           ht: Test new/updated files
       n - skip new filename prompt            ht0: using CRC32
       o - skip overwrite file query           ht1: using compare
       t - skip overwrite directory        hx: use recovery records
           attributes query                hy: set security envelope
       v - skip proceed to next volume         (registered version only)
           query                           hz: comment chapters
       jycn: skip create, new name             hzC.CMT: use C.CMT for comments
             queries                           hzNUL: use to strip comments
   jz: supply file for file comment
       jzF.CMT: use F.CMT for comments
       jzNUL: use to strip comments

                                                 path storing style

                                                 Chapter" command

                                                 engine (Win32 version only)

                                                 specified command

                                                 output

HighlightColor                       0..15       Highlight color

Environment variable setting:
-----------------------------

JAR_CFG_FILE=c:\exe\jar.cfg
JAR_CFG_SECTION=Special

JAR errorlevels:
----------------

0 ->  success
1 ->  warning
2 ->  fatal error
3 ->  CRC error
5 ->  disk full or write error
6 ->  cannot open archive
7 ->  simple user error (bad parameters)
8 ->  not enough memory
9 ->  not a JAR archive
10 -> internal error
11 -> Ctrl+C pressed
